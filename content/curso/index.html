    Curso de Fundamentos de JavaScript

    Este curso de Fundamentos de JavaScript está diseñado para principiantes sin experiencia previa en programación, con un enfoque práctico, interactivo y orientado a proyectos. Consta de 10 clases de 6 horas cada una, impartidas por El SENA, con el objetivo de capacitar a los estudiantes para desarrollar aplicaciones web funcionales utilizando JavaScript moderno (ES6+). A continuación, se detalla el plan completo, incluyendo objetivos, temas, metodología, actividades, distribución del tiempo, recursos y consideraciones adicionales.

    Clase 1: Introducción a JavaScript y Primeros Pasos

    Objetivos

    • Comprender qué es JavaScript y su rol en el desarrollo web.
    • Configurar un entorno de desarrollo funcional.
    • Escribir y ejecutar scripts básicos en JavaScript.

    Temas

    • Introducción a JavaScript: historia, usos y aplicaciones.
    • Configuración del entorno: Visual Studio Code, Node.js, extensiones (Live Server, Prettier).
    • Primeros scripts: uso de console.log, alert.
    • Estructura básica de un proyecto web: HTML, CSS, JavaScript.

    Metodología

    • Teoría (1.5 horas): Presentación sobre JavaScript y demostración de configuración del entorno.
    • Práctica (3.5 horas): Ejercicios guiados y laboratorio para crear scripts.
    • Revisión (1 hora): Resolución de dudas, discusión grupal y retroalimentación.

    Actividades Prácticas
    1. Instalar y configurar VS Code y Node.js.
    2. Crear un script que imprima "¡Hola, mundo!" en la consola y en una ventana emergente.
    3. Modificar un archivo HTML para incluir un script JavaScript que muestre un mensaje.
    4. Mini-proyecto: Crear una página web que muestre un saludo personalizado basado en un nombre ingresado en un prompt.

    Distribución del Tiempo

    • Teoría: 1.5 horas (presentación y demo).
    • Práctica: 3.5 horas (configuración: 1 hora, ejercicios: 1.5 horas, mini-proyecto: 1 hora).
    • Revisión: 1 hora (dudas y retroalimentación).

    Recursos

    • Herramientas: Visual Studio Code, Node.js, navegador (Chrome/Firefox).
    • Materiales: Diapositivas introductorias, guía de instalación en PDF, plantilla HTML/CSS.
    • Referencias: MDN Web Docs - JavaScript, JavaScript.info - Introducción.

    Clase 2: Sintaxis y Operaciones Básicas

    Objetivos

    • Dominar los tipos de datos y operadores básicos en JavaScript.
    • Utilizar entrada de usuario con prompt.
    • Crear scripts interactivos en la consola.

    Temas

    • Tipos de datos: string, number, boolean, null, undefined.
    • Declaración de variables: let, const, var.
    • Operadores: aritméticos (+, -, *, /, %), comparación (==, ===, !=), lógicos (&&, ||, !).
    • Entrada de usuario con prompt y conversión de tipos.

    Metodología

    • Teoría (1.5 horas): Explicación de tipos de datos y operadores con ejemplos en vivo.
    • Práctica (3.5 horas): Ejercicios guiados y laboratorio para manipular variables.
    • Revisión (1 hora): Corrección de ejercicios y discusión grupal.

    Actividades Prácticas
    1. Declarar variables de diferentes tipos y mostrarlas en la consola.
    2. Crear un programa que realice operaciones aritméticas con dos números ingresados por el usuario.
    3. Comparar dos valores ingresados con prompt y mostrar si son iguales o diferentes.
    4. Mini-proyecto: Calculadora interactiva en la consola que realice suma, resta, multiplicación y división según la elección del usuario.

    Distribución del Tiempo

    • Teoría: 1.5 horas (explicación y ejemplos).
    • Práctica: 3.5 horas (ejercicios: 2 horas, mini-proyecto: 1.5 horas).
    • Revisión: 1 hora (corrección y dudas).

    Recursos

    • Herramientas: VS Code, Node.js, navegador.
    • Materiales: Diapositivas sobre sintaxis, ejemplos de código comentados, guía de operadores.
    • Referencias: MDN - Tipos de datos, JavaScript.info - Variables.

    Clase 3: Control de Flujo

    Objetivos

    • Implementar estructuras condicionales y bucles para controlar el flujo de un programa.
    • Resolver problemas lógicos con estructuras de control.
    • Crear programas interactivos con lógica condicional.

    Temas

    • Condicionales: if, else if, else, switch.
    • Bucles: for, while, do-while.
    • Instrucciones: break, continue.
    • Anidación de condicionales y bucles.

    Metodología

    • Teoría (1.5 horas): Explicación de estructuras de control con diagramas y ejemplos en vivo.
    • Práctica (3.5 horas): Ejercicios guiados y laboratorio colaborativo.
    • Revisión (1 hora): Resolución de ejercicios y discusión en grupo.

    Actividades Prácticas
    1. Crear un programa que determine si un número es par o impar usando if.
    2. Usar un bucle for para imprimir los números del 1 al 10.
    3. Implementar un juego de adivinar un número (el usuario ingresa un número hasta acertar).
    4. Mini-proyecto: Generador de tablas de multiplicación que muestre la tabla de un número ingresado por el usuario.

    Distribución del Tiempo

    • Teoría: 1.5 horas (explicación y demo).
    • Práctica: 3.5 horas (ejercicios: 2 horas, mini-proyecto: 1.5 horas).
    • Revisión: 1 hora (corrección y dudas).

    Recursos

    • Herramientas: VS Code, navegador.
    • Materiales: Diapositivas sobre control de flujo, ejemplos de código, guía de bucles.
    • Referencias: MDN - Estructuras de control, JavaScript.info - Bucles.

    Clase 4: Funciones y Reutilización

    Objetivos

    • Definir y utilizar funciones para modularizar código.
    • Comprender el concepto de parámetros y retorno.
    • Aplicar funciones en problemas prácticos.

    Temas

    • Declaración de funciones: function, expresiones de función, arrow functions.
    • Parámetros: valores por defecto, rest parameters.
    • Retorno de valores y ámbito (scope).
    • Llamadas a funciones y reutilización de código.

    Metodología

    • Teoría (1.5 horas): Explicación de funciones con ejemplos prácticos.
    • Práctica (3.5 horas): Ejercicios guiados y laboratorio en grupos pequeños.
    • Revisión (1 hora): Corrección de ejercicios y retroalimentación.

    Actividades Prácticas

    1. Crear una función que calcule el cuadrado de un número.
    2. Escribir una función que valide si una cadena es vacía o no.
    3. Implementar una función que reciba dos números y devuelva el mayor.
    4. Mini-proyecto: Conversor de unidades (temperatura: Celsius a Fahrenheit, o moneda: pesos a dólares) usando funciones.

    Distribución del Tiempo

    • Teoría: 1.5 horas (explicación y demo).
    • Práctica: 3.5 horas (ejercicios: 2 horas, mini-proyecto: 1.5 horas).
    • Revisión: 1 hora (corrección y dudas).

    Recursos

    • Herramientas: VS Code, navegador.
    • Materiales: Diapositivas sobre funciones, ejemplos de código, guía de parámetros.
    • Referencias: MDN - Funciones, JavaScript.info - Funciones.

    Clase 5: Estructuras de Datos: Arreglos

    Objetivos

    • Manipular arreglos para almacenar y procesar datos.
    • Utilizar métodos de arreglos para operaciones comunes.
    • Crear programas que gestionen listas de datos.

    Temas

    • Creación y acceso a arreglos.
    • Métodos: push, pop, shift, unshift, slice, forEach, map.
    • Iteración sobre arreglos: bucles y métodos funcionales.
    • Arreglos multidimensionales (básico).

    Metodología

    • Teoría (1.5 horas): Explicación de arreglos y métodos con ejemplos en vivo.
    • Práctica (3.5 horas): Ejercicios guiados y laboratorio colaborativo.
    • Revisión (1 hora): Resolución de ejercicios y discusión grupal.

    Actividades Prácticas

    1. Crear un arreglo de nombres y mostrarlos en la consola usando forEach.
    2. Filtrar un arreglo de números para mostrar solo los mayores a 10.
    3. Usar map para duplicar los valores de un arreglo numérico.
    4. Mini-proyecto: Lista de compras con funciones CRUD (crear, leer, actualizar, eliminar) usando arreglos.

    Distribución del Tiempo

    • Teoría: 1.5 horas (explicación y demo).
    • Práctica: 3.5 horas (ejercicios: 2 horas, mini-proyecto: 1.5 horas).
    • Revisión: 1 hora (corrección y dudas).

    Recursos

    • Herramientas: VS Code, navegador.
    • Materiales: Diapositivas sobre arreglos, ejemplos de código, guía de métodos.
    • Referencias: MDN - Arreglos, JavaScript.info - Arreglos.

    Clase 6: Estructuras de Datos: Objetos

    Objetivos

    • Crear y manipular objetos para estructurar datos complejos.
    • Comprender el formato JSON y su uso.
    • Desarrollar aplicaciones que gestionen datos estructurados.

    Temas

    • Creación de objetos: propiedades, métodos.
    • Acceso a propiedades: notación de punto y corchetes.
    • Objetos anidados y JSON.
    • Iteración sobre objetos: for...in, Object.keys.

    Metodología

    • Teoría (1.5 horas): Explicación de objetos y JSON con ejemplos prácticos.
    • Práctica (3.5 horas): Ejercicios guiados y laboratorio en grupos.
    • Revisión (1 hora): Corrección de ejercicios y retroalimentación.

    Actividades Prácticas

    1. Crear un objeto que represente un estudiante (nombre, edad, notas).
    2. Modificar propiedades de un objeto basado en entrada del usuario.
    3. Convertir un objeto a JSON y mostrarlo en la consola.
    4. Mini-proyecto: Simulador de agenda personal que permita agregar y mostrar eventos (fecha, descripción).

    Distribución del Tiempo

    • Teoría: 1.5 horas (explicación y demo).
    • Práctica: 3.5 horas (ejercicios: 2 horas, mini-proyecto: 1.5 horas).
    • Revisión: 1 hora (corrección y dudas).

    Recursos

    • Herramientas: VS Code, navegador.
    • Materiales: Diapositivas sobre objetos, ejemplos de código, guía de JSON.
    • Referencias: MDN - Objetos, JavaScript.info - Objetos.

    Clase 7: Introducción al DOM

    Objetivos

    • Acceder y modificar elementos HTML mediante el DOM.
    • Crear interfaces web dinámicas.
    • Integrar JavaScript con HTML y CSS.

    Temas

    • Qué es el DOM y su estructura.
    • Métodos: getElementById, querySelector, querySelectorAll.
    • Modificación: innerHTML, textContent, style.
    • Creación y eliminación de elementos: createElement, appendChild, remove.

    Metodología

    • Teoría (1.5 horas): Explicación del DOM con demostraciones en vivo.
    • Práctica (3.5 horas): Ejercicios guiados y laboratorio colaborativo.
    • Revisión (1 hora): Resolución de ejercicios y discusión grupal.

    Actividades Prácticas
    1. Cambiar el texto de un elemento HTML usando innerHTML.
    2. Modificar el color de fondo de una página al hacer clic en un botón.
    3. Crear una lista dinámica a partir de un arreglo.
    4. Mini-proyecto: Generador de tarjetas dinámicas que muestre información de productos (nombre, precio, descripción).

    Distribución del Tiempo

    • Teoría: 1.5 horas (explicación y demo).
    • Práctica: 3.5 horas (ejercicios: 2 horas, mini-proyecto: 1.5 horas).
    • Revisión: 1 hora (corrección y dudas).

    Recursos

    • Herramientas: VS Code, navegador.
    • Materiales: Diapositivas sobre el DOM, plantilla HTML/CSS, ejemplos de código.
    • Referencias: MDN - DOM, JavaScript.info - DOM.

    Clase 8: Manejo de Eventos

    Objetivos

    • Implementar eventos para crear aplicaciones interactivas.
    • Utilizar addEventListener para manejar interacciones del usuario.
    • Validar entradas en formularios dinámicos.

    Temas

    • Introducción a eventos: click, input, submit, keydown.
    • Uso de addEventListener y funciones callback.
    • Propagación de eventos: bubbling y capturing (básico).
    • Prevención de comportamiento por defecto: preventDefault.

    Metodología

    • Teoría (1.5 horas): Explicación de eventos con ejemplos en vivo.
    • Práctica (3.5 horas): Ejercicios guiados y laboratorio en grupos pequeños.
    • Revisión (1 hora): Corrección de ejercicios y retroalimentación.

    Actividades Prácticas
    1. Crear un botón que muestre un mensaje al hacer clic.
    2. Implementar un campo de texto que muestre su contenido en tiempo real.
    3. Validar un formulario para asegurar que no esté vacío.
    4. Mini-proyecto: Formulario dinámico con validación en tiempo real (ej. verificar longitud de contraseña, formato de correo).

    Distribución del Tiempo

    • Teoría: 1.5 horas (explicación y demo).
    • Práctica: 3.5 horas (ejercicios: 2 horas, mini-proyecto: 1.5 horas).
    • Revisión: 1 hora (corrección y dudas).

    Recursos

    • Herramientas: VS Code, navegador.
    • Materiales: Diapositivas sobre eventos, plantilla HTML/CSS, ejemplos de código.
    • Referencias: MDN - Eventos, JavaScript.info - Eventos.

    Clase 9: Asincronía y Consumo de APIs

    Objetivos

    • Comprender los conceptos de asincronía en JavaScript.
    • Consumir APIs públicas usando fetch y async/await.
    • Mostrar datos dinámicamente en una página web.

    Temas

    • Introducción a asincronía: callbacks, promesas, async/await.
    • Uso de fetch para realizar solicitudes HTTP.
    • Manejo de errores con try/catch.
    • Consumo de APIs públicas: PokeAPI, OpenWeather.

    Metodología

    • Teoría (1.5 horas): Explicación de asincronía y APIs con demostraciones en vivo.
    • Práctica (3.5 horas): Ejercicios guiados y laboratorio colaborativo.
    • Revisión (1 hora): Resolución de ejercicios y discusión grupal.

    Actividades Prácticas
    1. Crear una función asíncrona que espere 2 segundos antes de mostrar un mensaje.
    2. Consumir PokeAPI para mostrar el nombre de un Pokémon en la consola.
    3. Mostrar una lista de datos de una API en una tabla HTML.
    4. Mini-proyecto: Aplicación de clima (usando OpenWeather) o galería de Pokémon (usando PokeAPI) que muestre datos dinámicos.

    Distribución del Tiempo

    • Teoría: 1.5 horas (explicación y demo).
    • Práctica: 3.5 horas (ejercicios: 2 horas, mini-proyecto: 1.5 horas).
    • Revisión: 1 hora (corrección y dudas).

    Recursos

    • Herramientas: VS Code, navegador.
    • Materiales: Diapositivas sobre asincronía, ejemplos de código, guía de fetch.
    • Referencias: MDN - Fetch API, JavaScript.info - Async/Await.

    Clase 10: Proyecto Final y Buenas Prácticas

    Objetivos

    • Integrar todos los conceptos aprendidos en un proyecto completo.
    • Aplicar buenas prácticas de codificación.
    • Presentar y evaluar proyectos en grupo.

    Temas

    • Repaso de conceptos: DOM, eventos, arreglos, objetos, APIs.
    • Buenas prácticas: nombrado claro, comentarios, modularización.
    • Organización de proyectos: estructura de archivos, reutilización de código.
    • Depuración básica: uso de la consola y debugger.

    Metodología

    • Teoría (1 hora): Repaso de conceptos y explicación de buenas prácticas.
    • Práctica (4 horas): Desarrollo del proyecto final en grupos pequeños.
    • Revisión (1 hora): Presentación de proyectos y retroalimentación grupal.

    Actividades Prácticas

    1. Refactorizar un código desordenado aplicando buenas prácticas.
    2. Crear un módulo reusable para una función común (ej. validación de formularios).
    3. Depurar un programa con errores usando la consola.

    4. Proyecto final: Desarrollo de una aplicación web, como:

      • Administrador de tareas (crear, editar, eliminar tareas).
      • Tienda simple (mostrar productos, agregar al carrito).
      • Blog interactivo (publicar y mostrar entradas).
      • La aplicación debe integrar DOM, eventos, arreglos, objetos y consumo de una API.

    Distribución del Tiempo

    • Teoría: 1 hora (repaso y buenas prácticas).
    • Práctica: 4 horas (ejercicios: 1 hora, proyecto final: 3 horas).
    • Revisión: 1 hora (presentaciones y retroalimentación).

    Recursos

    • Herramientas: VS Code, navegador.
    • Materiales: Diapositivas sobre buenas prácticas, plantilla de proyecto, rúbrica de evaluación.
    • Referencias: MDN - Buenas prácticas, JavaScript.info - Estructura de código.

    Entregables

    • Plan detallado: Documento Markdown con las 10 clases, incluyendo objetivos, temas, actividades, distribución del tiempo y recursos.
    • Materiales de apoyo:
      • Diapositivas para cada clase (formato PPT o PDF).
      • Ejemplos de código comentados (archivos JS, HTML, CSS).
      • Guías en PDF: instalación, sintaxis, DOM, eventos, APIs.
      • Enlaces a recursos externos: MDN Web Docs, JavaScript.info, freeCodeCamp.
    • Proyecto final:
      • Descripción del proyecto con requisitos claros (integrar DOM, eventos, arreglos, objetos, API).
      • Rúbrica de evaluación: funcionalidad (40%), código limpio (30%), creatividad (20%), presentación (10%).
    • Plantillas: Archivos base (HTML, CSS, JS) para ejercicios y proyectos.

    Consideraciones Adicionales

    • Ejercicios motivadores: Los proyectos están diseñados para ser relevantes (ej. aplicaciones para pequeñas empresas, herramientas educativas) y escalables según el nivel del estudiante.
    • Demostraciones en vivo: Cada clase incluye ejemplos prácticos ejecutados por el instructor para aclarar conceptos complejos como el DOM o la asincronía.
    • Trabajo colaborativo: Las clases 5 y 9 incluyen actividades en grupos pequeños para fomentar el aprendizaje en equipo.
    • Escalabilidad: Los mini-proyectos ofrecen opciones avanzadas (ej. agregar estilos CSS, incluir más validaciones) para estudiantes con mayor rapidez.
    • Plantillas de código: Se proporcionan archivos base para los principiantes, pero los ejercicios fomentan escribir código desde cero para reforzar el aprendizaje.
    • Evaluación formativa: Los ejercicios prácticos se revisan en cada clase, con autoevaluaciones opcionales para que los estudiantes reflexionen sobre su progreso.

    Rúbrica de Evaluación del Proyecto Final

    Criterio Peso Descripción
    Funcionalidad 40% La aplicación cumple con los requisitos (DOM, eventos, arreglos, objetos, API).
    Código limpio 30% Uso de nombres claros, comentarios, modularización y buenas prácticas.
    Creatividad 20% Diseño visual atractivo, funcionalidades adicionales o enfoques innovadores.
    Presentación 10% Explicación clara del proyecto, demostración funcional y respuestas a preguntas.
